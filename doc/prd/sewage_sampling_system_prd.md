## 1. 产品概述

### 1.1 背景与定位
本产品为**污水自动采样系统**的移动端 Android 控制与监控终端。该系统通过远程 MQTT 协议连接部署在污水处理现场的自动采样设备（树莓派控制器），同时支持通过 BLE 蓝牙广播直连现场水位传感器。
Android 客户端**不直接参与底层的物理控制与执行逻辑**（控制与执行逻辑在远端采样机内部运行），而是作为**人机交互、参数配置下发、状态实时监控、异常警报接收、远程拍照触发以及本地图文日志归档**的中心终端。

### 1.2 核心业务流程
1. 用户通过 App 配置远端设备的运行时间参数（药液卸载时长、水泵运行时间等）。
2. App 自动通过 MQTT 发布配置参数，远端设备接收后更新自身执行逻辑。
3. 远端设备在运行过程中，实时将三条药液管道的状态与现场拍摄的污水照片回传。
4. App 实时监测三条管道（次氯酸钠、碳源、铁盐）的有水/无水流状态，并展示 10 步采样流程进度。
5. App 即使在后台或锁屏状态下，也通过前台服务（Foreground Service）保持 MQTT 连接与 BLE 扫描，确保即时归档图文日志并弹出系统通知。
6. 用户可随时在本地沙盒文件系统中查询、筛选和浏览历史日志及污水照片。
7. 支持多站点部署：通过配置站点英文名切换 MQTT 主题前缀和图片过滤，实现一个 App 对接多套设备。

### 1.3 开发与编译环境说明
本项目开发与构建环境已完全在本地就绪，包含以下基础设施：
* **IDE 集成开发环境**：本地已安装 **Android Studio**，内置完整开发、构建及真机/模拟器调试工具链。
* **Android SDK 路径**：本地 SDK 目录为 `C:\Users\feng\AppData\Local\Android\Sdk`。
* **构建与编译工具**：使用原生 Gradle 构建系统进行代码编译与 APK 打包。
* **调试与日志查看**：配合 Android Studio Logcat 控制台或 `adb` 工具对常驻 MQTT Service 以及本地沙盒日志读写进行调试。

### 1.4 多站点部署机制
系统支持通过修改「本地配置」中的 **站点英文名**（如 `dongzhan`、`xizhan`）动态切换 MQTT 主题前缀：
* 所有订阅与发布主题中的 `{stationName}` 占位符会替换为站点英文名。
* 图片回传按站点英文名过滤，忽略其他站点的图片消息。
* **站点中文名**仅用于 App 顶部标题栏显示（如「济南东站污水厂」）。

## 2. 系统架构与数据流设计

### 2.1 软件架构 (MVVM + Repository)
系统采用现代化原生 Android 架构：
* **View (Jetpack Compose)**：声明式 UI 层，负责 Material Design 3 风格界面的呈现与交互。
* **ViewModel**：持有 UI 状态（StateFlow），向 View 暴露响应式数据，处理生命周期变化。
* **Repository (Data Layer)**：数据仓库，统一调度底层数据流：
  * **MQTT Client Service**：后台网络服务，负责 MQTT 消息收发。
  * **BLE Scanner**：蓝牙低功耗扫描，直接解析传感器广播包。
  * **Local File Manager**：本地沙盒文件管理器，负责 JSON Lines 日志写盘与图片保存。

### 2.2 MQTT 主题与数据格式规范

> **命名约定**：以下所有主题中 `{stationName}` 代表站点英文名（可配置，默认值 `dongzhan`）。

#### 2.2.1 【订阅】系统与设备运行状态主题
* **系统状态主题 (`water/{stationName}/system/status`)**：
  * **流向**：树莓派 → Android App
  * **载荷格式**：文本。指示当前工作状态，例如 `normal`。
* **系统诊断与运行信息主题 (`water/{stationName}/system/info`)**：
  * **流向**：树莓派 → Android App
  * **载荷格式**：使用换行符 `\n` 分割的三行文本。
  * **内容格式**：
    * 第一行：当前时间 (格式如 `06月04日 20:20`)
    * 第二行：开机启动时间 (格式如 `06月04日 20:00`)
    * 第三行：运行时间 (格式如 `0天 2小时 15分钟`)

#### 2.2.2 【订阅】通道远程日志主题：`water/{stationName}/log/+`
* **流向**：树莓派 → Android App
* **主题通配符**：`+` 对应通道号（即 `1`，`2`，`3` 通道）。
* **载荷格式**：使用换行符 `\n` 分割的三行文本（包含时间戳、通道编号、动作与报警消息）。
* **内容格式示例**：
  * 第一行：时间戳 (`%m月%d日 %H:%M`)，如 `06月04日 20:15`
  * 第二行：通道前缀，如 `通道1:`
  * 第三行：多事件合并消息，如 `检测触发:待稳 | 泵启:第1次`
* **App端处理逻辑**：
  * 接收到日志消息后，将其按通道号分类，并在本地进行持久化。
  * 根据关键词自动判断日志级别：包含 `ERROR`/`异常`/`失败` 标记为 ERROR，包含 `警告`/`丢` 标记为 WARN，其余为 INFO。
  * 若收到 WARN 或 ERROR 级别日志，即触发系统通知。

#### 2.2.3 【订阅】图片二进制数据主题：`water/photo/status/{stationName}`
* **流向**：树莓派 → Android App
* **载荷格式**：原始图片二进制字节流 (Raw byte array - JPG/PNG 格式)
* **大小限制**：单张图片不超过 5MB
* **站点过滤**：App 仅接受与当前配置站点名匹配的图片，其他站点的图片静默忽略
* **App端处理逻辑**：
  1. 接收到字节数组后，在本地沙盒 `images/` 目录下创建以时间戳命名的图片文件 `IMG_yyyyMMdd_HHmmss.jpg`。
  2. 在当前活跃通道的本地日志记录中，关联该图片的本地路径（即填充 `image_path` 字段）。

#### 2.2.4 【订阅】传感器原始数据主题：`water/sensor/status`
* **流向**：传感器节点 → Android App
* **载荷格式**：JSON 对象
* **内容格式**：
  ```json
  {"name": "dongzhan", "ch1": 12345, "ch2": 12300, "ch3": 12200, "ch4": 12100}
  ```
  * `name`：传感器节点标识（需与 App 当前激活的调试前缀匹配才处理）
  * `ch1`~`ch4`：四通道电容传感器原始读数（16位无符号整数）
* **App端处理逻辑**：
  1. 仅当 `name` 字段与当前调试激活的前缀一致时才处理。
  2. 物理通道映射：`ch4→通道1, ch3→通道2, ch2→通道3, ch1→通道4`（反序映射）。
  3. 对每个通道进行滑动均值滤波（窗口50）、基准线计算（窗口200）及施密特触发器判断。

#### 2.2.5 【发布】采样时间与水泵参数主题
App 通过往以下主题发布控制消息，以达到动态修改远端设备运行参数的目的：
* **采样通道预期时长配置主题 (`water/{stationName}/config/duration/+`)**：
  * **流向**：Android App → 树莓派 (QoS = 1)
  * **主题通配符**：`+` 对应要配置的通道号（即 `1`，`2`，`3`）。
  * **载荷格式**：预期时长数值（单位：分钟，字符串形式，如 `60.0` 或 `120`）。
* **通道水泵启动时长配置主题 (`water/{stationName}/config/pump_time/+`)**：
  * **流向**：Android App → 树莓派 (QoS = 1)
  * **主题通配符**：`+` 对应要配置的通道号（即 `1`，`2`，`3`）。
  * **载荷格式**：水泵运行秒数（单位：秒，字符串形式，如 `10.0` 或 `15`）。

#### 2.2.6 【发布】远程拍照触发主题：`water/photo/take`
* **流向**：Android App → 树莓派 (QoS = 1)
* **载荷格式**：站点英文名字符串（如 `dongzhan`），树莓派据此确定是否响应拍照指令。
* **App端处理逻辑**：
  * 用户点击「远程拍照」按钮后发送拍照指令，按钮进入 Loading 状态。
  * 若 15 秒内未收到图片回传，自动解锁按钮（防止丢包导致永久卡死）。

#### 2.2.7 【发布】传感器调试控制主题：`water/sensor/start`
* **流向**：Android App → 传感器节点 (QoS = 2)
* **载荷格式**：
  * 启动：站点英文名字符串（如 `dongzhan`），传感器节点收到后开始上报数据。
  * 停止：固定字符串 `stop`。

### 2.3 BLE 蓝牙广播数据协议
系统支持通过 BLE 广播直接接收传感器节点数据（不建立连接，仅扫描广播包）。
* **厂商自定义数据 ID**：`0xFFFF`
* **数据格式**（至少 8 字节，可选第 9 字节）：
  * Byte 0~1：通道0 原始值 (大端序 uint16)
  * Byte 2~3：通道1 原始值 (大端序 uint16)
  * Byte 4~5：通道2 原始值 (大端序 uint16)
  * Byte 6~7：通道3 原始值 (大端序 uint16)
  * Byte 8（可选）：序列号 (uint8)，用于去重
* **去重机制**：若序列号与上一次相同，丢弃该广播包。
* **后台保护**：
  * 获取 `PARTIAL_WAKE_LOCK` 防止 CPU 休眠。
  * 15 分钟无数据自动停止扫描并释放资源。

## 3. 功能需求

系统包含 **三个底部导航页**、**三个菜单隐藏页**、以及 **一个全屏弹窗调试页**，共 7 个独立功能视图。

### 3.1 主监控页 (Monitor Screen) — 底部导航 Tab 1
提供三段式核心监控视图：管道水流可视化、采样流程进度条、图像快照与网关诊断。

1. **药液管道水流状态卡片（置顶）**：
   * 三张并排卡片分别对应三条药液管道：**次氯酸钠**（通道1）、**碳源**（通道2）、**铁盐**（通道3）。
   * 每张卡片通过颜色与图标动态显示有水/无水状态：
     * 有水流：蓝色水滴图标 💧，主色容器高亮背景，蓝色描边。
     * 无水流：灰色圆点 ⚪，浅灰色底板，无描边。
   * 水流状态根据通道日志消息中的关键词自动推断（`泵启`/`水泵启动` → 有水，`泵停`/`排空`/`结束` → 无水）。

2. **10 步采样流程起伏式步骤条**：
   * 展示 10 个状态节点：`空闲 → 待稳 → 取头样 → 延时 → 取中样 → 延时 → 取尾样 → 延时 → 排空 → 结束`。
   * 使用起伏折线高度编排（采样步骤在高位，等待步骤在低位），形成波浪形视觉效果。
   * 当前激活步骤高亮蓝色，已完成步骤显示次级色，未到达步骤灰色。
   * 步骤索引通过 `system/status` 主题的返回值或日志关键词匹配自动更新。

3. **污水图像快照监视区**：
   * 展示最新接收到的现场污水采样照片（使用 Coil 异步加载）。
   * 照片下方标明照片回传接收的时间戳。
   * 点击照片可弹出全屏大图预览对话框，支持**多指缩放**（1x~5x）与**拖拽平移**。
   * 提供「📷 远程拍照」按钮，点击后向 `water/photo/take` 发送拍照指令，按钮进入 Loading 状态（最长 15 秒超时自动解锁）。
   * 若尚未接收到任何照片，显示空态占位提示。

4. **网关物理诊断看板**：
   * 实时解析展示 `system/info` 回传的三行系统信息：设备当前时钟、控制器启动时间、累积运行时长。
   * 右上角状态指示灯：通过 `system/status` 判断柜体在线（绿色）/ 离线（红色）。

### 3.2 参数设定页 (Settings Screen) — 底部导航 Tab 2
通过 Tab 切换配置三个通道的**药液卸载时间**和**水泵运行时间**，并一键远端同步。

1. **双 Tab 分类**：
   * **Tab 1「药液卸载时间」**：三通道的采样预期总时长配置。
   * **Tab 2「水泵运行时间」**：三通道每次水泵开启后的工作时间配置。

2. **通道参数卡片（每 Tab 3 张）**：
   每个通道卡片包含：
   * **通道标识徽章**：蓝色（次氯酸钠）、绿色（碳源）、橙色（铁盐）。
   * **微调滑块 (Slider)**：
     * 药液卸载时间：范围 `0 ~ 180` 分钟。
     * 水泵运行时间：范围 `0 ~ 60` 秒。
   * **实时数值显示**：滑块右侧实时显示当前值与单位。
   * 拖动滑块时，参数实时保存至 `SharedPreferences` 本地持久化。

3. **保存与同步**：
   * 底部提供「保存」大按钮。点击后将三个通道的全部参数通过 MQTT 向对应配置主题发布。
   * 同步结果通过 MD3 `Snackbar` 反馈：「保存成功！」或「保存失败！」。

### 3.3 日志查询页 (Logs Screen) — 底部导航 Tab 3
用于浏览、查询和筛选本地保存的所有历史数据和照片。

1. **筛选器栏**：
   * **日期选择按钮**：点击弹出系统 DatePicker，默认选中今日。
   * **搜索框**：支持按文字模糊搜索日志中的 `message` 字段。

2. **日志卡片列表**：
   * 使用 `LazyColumn` 高效渲染长列表。
   * 每张卡片包含：
     * **通道编号徽章**：`C1`(蓝) / `C2`(绿) / `C3`(橙)。
     * **时间戳** (`HH:mm:ss`)。
     * **日志内容文本**。
     * **级别颜色高亮**：INFO 默认背景、WARN 淡橘色、ERROR 淡红色。
   * **图文集成**：若本条日志关联了图片，卡片右侧以 50dp 微缩图形式展示照片，点击弹出大图预览对话框。

3. **空态提示**：当选定日期无日志记录时，显示「📂 该日暂无动作日志记录」。

### 3.4 MQTT 云端配置页 (Config Screen) — 菜单入口
配置用于与远端设备进行通信的底层 MQTT 连接参数。

1. **连接参数表单**：
   * **服务器地址**（文本输入框，支持 IP 或域名）。
   * **端口号**（数字输入框，如 1883 / 8883）。
   * **客户端 ID**（为空时自动生成 UUID）。
   * **认证用户名**（文本输入框）。
   * **密码**（密码输入框，支持一键切换明文/密文显示）。
   * 所有配置保存至 `EncryptedSharedPreferences` 加密存储。

2. **连接控制**：
   * 提供「保存并建立连接」/「应用并重启连接」按钮（文案根据当前连接状态动态切换）。
   * 点击后参数持久化并通过 MqttBus 发送 `Reconnect` 指令，触发服务端重连。

### 3.5 网络诊断日志页 (Network Logs Screen) — 菜单入口
独立页面，显示底层的 MQTT 通信和重连日志。

1. **高对比度深色控制台**（背景色 `#1E1E1E`），只读滚动显示 MQTT 客户端日志。
2. **关键词着色**：`连接成功` 绿色、`失败/丢失/断开` 红色、其余信息浅灰色。
3. **最大缓存 100 条**，新日志从顶部插入。
4. 使用等宽字体 (`FontFamily.Monospace`)，模拟命令行终端质感。

### 3.6 本地站点配置页 (Station Settings Screen) — 菜单入口
设置本地运行参数，包括站点标识与通知偏好。

1. **站点标识配置**：
   * **站点英文名**（设备标识 / MQTT 主题前缀，如 `dongzhan`）。
   * **站点中文名**（界面全局标题，如「济南东站污水厂」）。

2. **通知偏好开关**：
   * **弹出通知消息框**：控制是否在异常报警时弹出系统通知。
   * **使用声音通知**：控制报警通知是否伴随铃声。

3. **GPS 定位开关**：控制 BLE 传感器调试时是否申请 GPS 定位权限。

4. **安全锁定机制**：当传感器调试处于运行状态时，所有配置项自动锁定不可编辑，页面顶部显示红色警告卡片。

5. **保存生效**：点击「保存配置」后，配置写入 `SharedPreferences` 并实时更新全局标题栏。

### 3.7 传感器调试窗口 (Sensor Debug Screen) — 全屏弹窗
全屏 Dialog，提供 BLE 广播和 MQTT 网络两种数据通道的传感器实时波形调试。

1. **双模式 Tab 切换**：
   * **BLE 广播模式**：通过 BLE 后台扫描直接接收传感器广播包，需要蓝牙和定位权限。进入页面自动启动扫描，离开自动停止。
   * **MQTT 网络模式**：通过 MQTT 远程接收传感器数据，提供手动「开始调试」/「停止调试」按钮。

2. **四通道选择器**：`Ch4`、`Ch3`、`Ch2`、`Ch1` 四个 RadioButton，切换查看不同物理通道。每个通道旁有状态指示灯（绿=无水，红=有水）。

3. **实时水状态显示**：页面中部显示当前选中通道的水位状态（「检测到液体 (有水)」红色 / 「干燥正常 (无水)」绿色）。

4. **实时数值面板**：四列数据卡片：
   * 原始值（金色）→ 滤波值（绿色）→ 基准线（蓝色）→ 触发阈值（红色）。

5. **历史波形曲线**：Canvas 绘制的四线折线图（原始/滤波/基准/阈值），最大显示近 N 个采样点（N = 屏幕宽度像素数）。

6. **前台防休眠**：调试界面打开期间自动设置 `FLAG_KEEP_SCREEN_ON`，关闭时自动恢复。

7. **MQTT 15 分钟超时保护**：离开调试界面后，若 MQTT 调试仍在运行，将在 15 分钟后自动停止，防止资源泄露。

## 4. 非功能性需求与性能指标

### 4.1 后台常驻服务与连接维持 (Foreground Service)
* **前台服务绑定**：App 必须在后台运行时以 **Foreground Service** 形式维持 MQTT 客户端实例和 BLE 扫描，并在状态栏显示常驻通知。
* **Android 14 适配**：使用 `FOREGROUND_SERVICE_TYPE_SPECIAL_USE` 类型声明，附带 `PROPERTY_SPECIAL_USE_FGS_SUBTYPE` 元数据说明。
* **自动重连机制**：在网络抖动或切换时，采用指数避退重连（时间间隔为 2, 4, 8, 16 ... 60 秒），避免在无网区域频繁重复连接导致耗电与发热。
* **通知推送**：在后台接收到等级为 `WARN` 或 `ERROR` 的日志时，通过系统高优先级通知渠道发送通知。通知行为可在「本地配置」中关闭或静音。

### 4.2 本地沙盒存储规范 (Scoped Storage)
* **免除权限申请**：数据统一存储在 `getExternalFilesDir(null)` 私有沙盒下，遵守 Android 10+ 分区存储规范，无需申请敏感的 `WRITE_EXTERNAL_STORAGE` 读写权限。
* **日志存储格式**：使用 **JSON Lines (JSONL)** 格式按天保存文件，文件名命名为 `log_yyyyMMdd.json`。每行数据写入时必须追加换行符，便于高效的流式文件读取。
* **日志条目格式**：
  ```json
  {"time":"20:15:30","channel":1,"level":"INFO","message":"泵启:第1次","image_path":"images/IMG_20260604_201530.jpg"}
  ```
* **图片存储**：保存至沙盒 `images/` 目录，文件名为 `IMG_yyyyMMdd_HHmmss.jpg`。
* **文件自动清理**：提供最大存储时长配置（默认保存近 30 天日志和图片），超过天数的历史文件在每日 App 启动时自动静默清理，防止撑爆用户手机存储。

### 4.3 敏感数据安全存储
* MQTT 连接密码使用 `EncryptedSharedPreferences` (AES256-GCM) 加密保存。
* 内置异常降级机制：若 Android Keystore 异常导致加密不可用，自动降级为普通 `SharedPreferences`。

### 4.4 UI 视觉与交互体验 (Material Design 3)
* **组件规范**：全面使用 MD3 组件库（`NavigationBar`, `Card`, `Slider`, `DatePicker`, `Snackbar`, `TopAppBar` 等）。
* **深浅色模式**：系统完美支持 Android 深浅色模式自适应切换。Android 12+ 使用动态配色（Dynamic Color），低版本使用 Purple 系静态配色。
* **横竖屏适配**：
  * 竖屏：标准底部导航栏。
  * 横屏：自定义 44dp 极窄侧边导航栏（规避 MD3 NavigationRail 的 56dp 限制），TopAppBar 高度压缩至 36dp。
* **启动画面**：3 秒品牌 Splash Screen（主色背景 + 应用图标 + 名称）。
* **开机自启动**：注册 `BOOT_COMPLETED` 广播接收器，开机后自动启动 MQTT 前台服务。

### 4.5 指令通信解耦机制 (MqttBus)
* ViewModel 与 MqttService 之间通过 `MqttBus` (内存单例 `SharedFlow`，缓冲容量128) 传递指令，替代传统 `Intent` 传参机制。
* 支持的指令类型：`Publish`、`Subscribe`、`Unsubscribe`、`Reconnect`。

## 5. 验证与测试计划

### 5.1 模拟端测试
1. **MQTT 代理环境**：本地部署 EMQX 代理，或使用公共测试代理（如 `broker.hivemq.com`）。
2. **设备端模拟器**：编写 Python / Node.js 脚本模拟采样设备，负责定时向 `water/{stationName}/log/1` 等主题发送模拟报文，以及向 `water/photo/status/{stationName}` 发送随机 JPG 二进制数据。

### 5.2 核心功能用例
1. **前后台持久化测试**：App 处于后台，模拟树莓派在通道日志主题发布报警消息，检查 Android 状态栏是否能正常弹出 Notification，并在重新打开 App 后能在日志列表中完整看到该条图文信息。
2. **离线重连测试**：在 App 运行中开启手机飞行模式，观察日志控制台是否准确打印重连等待日志。关闭飞行模式后，观察是否在 10 秒内自动恢复连接。
3. **参数同步一致性测试**：在参数设置页修改通道 1 的「水泵运行时间」为 `45` 秒，点击保存，观察树莓派在 `water/{stationName}/config/pump_time/1` 主题接收到的负载值是否为 `45.0`。
4. **远程拍照测试**：点击远程拍照按钮，验证按钮进入 Loading 状态，树莓派端收到拍照指令并回传图片后，App 端自动显示最新照片。
5. **多站点切换测试**：修改站点英文名，验证 MQTT 主题前缀自动切换、图片过滤按新站点名生效。
6. **BLE 传感器调试测试**：进入传感器调试界面 BLE 模式，验证广播数据接收、四通道波形图绘制、有水/无水状态判断。
7. **MQTT 调试超时测试**：在 MQTT 调试模式启动后离开调试界面，15 分钟后验证调试自动停止。
