## 1. 产品概述

### 1.1 背景与定位
本产品为**污水自动采样系统**的移动端 Android 控制与监控终端。该系统通过远程 MQTT 协议连接部署在污水处理现场的自动采样设备。
Android 客户端**不直接参与底层的物理控制与执行逻辑**（控制与执行逻辑在远端采样机内部运行），而是作为**人机交互、参数配置下发、状态实时监控、异常警报接收以及本地图文日志归档**的中心终端。

### 1.2 核心业务流程
1. 用户通过 App 配置远端设备的运行时间参数（取样总时间、加药时间、水泵时间等）。
2. App 自动通过 MQTT 发布配置参数，远端设备接收后更新自身执行逻辑。
3. 远端设备在运行过程中，实时将状态数值（pH 值、温度、浊度等）与现场拍摄的污水照片回传。
4. App 即使在后台或锁屏状态下，也通过前台服务（Foreground Service）保持 MQTT 连接，确保即时归档图文日志并弹出系统通知。
5. 用户可随时在本地沙盒文件系统中查询、筛选和浏览历史日志及污水照片。

## 2. 系统架构与数据流设计

### 2.1 软件架构 (MVVM + Repository)
系统采用现代化原生 Android 架构：
* **View (Jetpack Compose)**：声明式 UI 层，负责 Material Design 3 风格界面的呈现与交互。
* **ViewModel**：持有 UI 状态（StateFlow），向 View 暴露响应式数据，处理生命周期变化。
* **Repository (Data Layer)**：数据仓库，统一调度底层数据流：
  * **MQTT Client Service**：后台网络服务，负责 MQTT 消息收发。
  * **Local File Manager**：本地沙盒文件管理器，负责 JSON Lines 日志写盘与图片保存。

### 2.2 MQTT 主题与数据格式规范
系统采用与 `pi_water` 项目完全一致的主题与格式定义，订阅和发布以下主题：

#### 2.2.1 【订阅】系统与设备运行状态主题
* **系统状态主题 (`pi_water/system/status`)**：
  * **流向**：树莓派 $\rightarrow$ Android App
  * **载荷格式**：文本。指示当前工作状态，例如 `normal`。
* **系统诊断与运行信息主题 (`pi_water/system/info`)**：
  * **流向**：树莓派 $\rightarrow$ Android App
  * **载荷格式**：使用换行符 `\n` 分割的三行文本。
  * **内容格式**：
    * 第一行：当前时间 (格式如 `06月04日 20:20`)
    * 第二行：开机启动时间 (格式如 `06月04日 20:00`)
    * 第三行：运行时间 (格式如 `0天 2小时 15分钟`)

#### 2.2.2 【订阅】通道远程日志主题：`pi_water/log/+`
* **流向**：树莓派 $\rightarrow$ Android App
* **主题通配符**：`+` 对应通道号（即 `1`，`2`，`3` 通道）。
* **载荷格式**：使用换行符 `\n` 分割的三行文本（包含时间戳、通道编号、动作与报警消息）。
* **内容格式示例**：
  * 第一行：时间戳 (`%m月%d日 %H:%M`)，如 `06月04日 20:15`
  * 第二行：通道前缀，如 `通道1:`
  * 第三行：多事件合并消息，如 `检测触发:待稳 | 泵启:第1次`
* **App端处理逻辑**：
  * 接收到日志消息后，将其按通道号分类，并在本地进行持久化。若收到包含 `ERROR` 或 `警告` 字样的日志且 App 处于后台，则触发系统通知。

#### 2.2.3 【订阅】图片二进制数据主题：`pi_water/photo`
* **流向**：树莓派 $\rightarrow$ Android App
* **载荷格式**：原始图片二进制字节流 (Raw byte array - JPG/PNG 格式)
* **App端处理逻辑**：
  1. 接收到字节数组后，在本地沙盒 `images/` 目录下创建以时间戳命名的图片文件 `IMG_yyyyMMdd_HHmmss.jpg`。
  2. 在当前活跃通道的本地日志记录中，关联该图片的本地路径（即填充 `image_path` 字段）。

#### 2.2.4 【发布】采样时间与水泵参数主题
App 通过往以下主题发布控制消息，以达到动态修改远端设备运行参数的目的：
* **采样通道预期时长配置主题 (`pi_water/config/duration/+`)**：
  * **流向**：Android App $\rightarrow$ 树莓派 (QoS = 0 / 1)
  * **主题通配符**：`+` 对应要配置的通道号（即 `1`，`2`，`3`）。
  * **载荷格式**：预期时长数值（单位：分钟，字符串形式，如 `60.0` 或 `120`）。
* **通道水泵启动时长配置主题 (`pi_water/config/pump_time/+`)**：
  * **流向**：Android App $\rightarrow$ 树莓派 (QoS = 0 / 1)
  * **主题通配符**：`+` 对应要配置的通道号（即 `1`，`2`，`3`）。
  * **载荷格式**：水泵运行秒数（单位：秒，字符串形式，如 `10.0` 或 `15`）。

## 3. 功能需求

系统包含四个独立的导航页面，各页面功能如下：

### 3.1 主监控页 (Monitor Screen)
提供实时的状态概览、通道日志监视以及诊断运行信息看板。
1. **连接状态卡片**：
   * 动态显示与远端 MQTT Broker 的连接状态（在线/绿灯，离线/灰灯，异常/红灯）。
   * 显示默认连接的代理地址 `voicevon.vicp.io:1883` 及状态。
2. **系统诊断看板**：
   * 实时解析展示订阅 `pi_water/system/info` 得到的系统诊断信息，包括：设备当前时间、设备开机启动时间、设备运行累计时长（展示为三行清晰卡片）。
   * 显示订阅 `pi_water/system/status` 得到的当前系统运行模式（如 `normal`）。
3. **分通道日志流实时监视**：
   * 分为三个选项卡或三个卡片区域，分别实时显示 **通道1**、**通道2**、**通道3** 订阅 `pi_water/log/1`、`/2`、`/3` 接收到的最新三行日志（时间、通道号、动作状态）。
4. **图像监视区**：
   * 展示最新订阅 `pi_water/photo` 接收到的现场污水采样照片。
   * 照片下方标明照片回传接收的时间戳。

### 3.2 日志查询页 (Logs Screen)
用于浏览、查询和筛选本地保存的所有历史数据和照片。
1. **筛选器栏**：
   * **日期选择组件**：点击弹出 MD3 日历，默认选中今日。
   * **搜索框**：支持按文字模糊搜索日志中的 `message` 字段。
2. **日志卡片列表**：
   * 每张卡片包含：时间 (`HH:mm:ss`)、日志级别（INFO/WARN/ERROR，带不同颜色高亮）、日志文本。
   * **图文集成**：若本条日志在回传时关联了图片，卡片右侧以微缩图形式展示照片。点击微缩图可弹出大图预览对话框，支持多指放大与拖拽。
3. **报文详情弹窗**：
   * 点击日志卡片任意空白处，弹出 Dialog 显示完整的底层 JSON 报文，供维护人员排查问题，支持一键复制。

### 3.3 MQTT 通信设置页 (MQTT Config Screen)
配置用于与远端设备进行通信的底层 MQTT 连接参数。
1. **连接参数表单**：
   * **服务器地址**（文本输入框，支持 IP 或域名，带格式校验）。
   * **端口号**（数字输入框，如 1883 / 8883）。
   * **客户端 ID**（自动生成 UUID，支持手动重置）。
   * **认证信息**（用户名和密码输入框，密码框支持一键切换明文/密文显示）。
2. **连接控制**：
   * 提供“保存并连接”按钮，点击后参数持久化至本地，并重启前台服务重连。
3. **网络日志控制台**：
   * 页面下方内嵌一个高对比度深色背景的控制台视窗，只读显示 MQTT 客户端连接日志（如 `Connecting...` $\rightarrow$ `Connected` $\rightarrow$ `Disconnected` 等），便于当场调试。

### 3.4 参数设置页 (Parameter Settings Screen)
设置三个通道的预期时长和水泵动作时间，并将其同步到远端树莓派中。
1. **参数配置卡片组**：
   页面按 **通道1**、**通道2**、**通道3** 进行分组配置。每个通道包含以下两个参数，配备微调滑块 (Slider) 和数值输入框 (TextField)：
   * **通道预期总时长 (expected_duration)**：
     * 用于控制该通道采样流程预期总时长，滑块范围：1 ~ 240 分钟。
     * 修改并点击保存后，自动发布至主题 `pi_water/config/duration/[通道号]`（例如：配置通道1，发布至 `pi_water/config/duration/1`，载荷为数值字符串）。
   * **通道水泵启动时长 (pump_work_time)**：
     * 用于配置每次水泵开启后的工作时间，滑块范围：1 ~ 300 秒。
     * 修改并点击保存后，自动发布至主题 `pi_water/config/pump_time/[通道号]`（例如：配置通道1，发布至 `pi_water/config/pump_time/1`，载荷为数值字符串）。
2. **保存与同步控制**：
   * 每个通道或整体提供一个“同步到设备”大按钮。点击后，参数保存至 `SharedPreferences` 本地持久化，并自动向对应的配置主题发布最新值。
   * 屏幕底部弹出 MD3 `Snackbar` 确认：“通道配置保存成功，正在同步至远端树莓派...”。

## 4. 非功能性需求与性能指标

### 4.1 后台常驻服务与连接维持 (Foreground Service)
* **前台服务绑定**：App 必须在后台运行时以 **Foreground Service** 形式维持 MQTT 客户端实例，并在状态栏显示常驻通知。
* **自动重连机制**：在网络抖动或切换时，采用指数避退重连（时间间隔为 $2, 4, 8, 16 \dots 60$ 秒），避免在无网区域频繁重复连接导致耗电与发热。
* **通知推送**：在后台接收到等级为 `WARN` 或 `ERROR` 的日志，或者接收到新图片时，若 App 处于后台，应通过系统通道发送高优先级 Notification。

### 4.2 本地沙盒存储规范 (Scoped Storage)
* **免除权限申请**：数据统一存储在 `getExternalFilesDir(null)` 私有沙盒下，遵守 Android 10+ 分区存储规范，无需申请敏感的 `WRITE_EXTERNAL_STORAGE` 读写权限。
* **日志存储格式**：使用 **JSON Lines (JSONL)** 格式按天保存文件，文件名命名为 `log_yyyyMMdd.json`。每行数据写入时必须追加换行符，便于高效的流式文件读取。
* **文件自动清理**：提供最大存储时长配置（默认保存近 30 天日志和图片），超过天数的历史文件在每日 App 启动时自动静默清理，防止撑爆用户手机存储。

### 4.3 UI 视觉与交互体验 (Material Design 3)
* **组件规范**：全面使用 MD3 组件库（`NavigationBar`, `Card`, `Slider`, `DatePicker`, `Snackbar` 等）。
* **深浅色模式**：系统必须完美支持 Android 深浅色模式的自适应切换，各主色调（Primary, Secondary, Container）使用动态配色，防止在刺眼阳光或昏暗室内出现高对比度视觉疲劳。
* **动效**：按钮点击水波纹效果，底部导航切换采用平滑的淡入淡出（Crossfade）微动效。

## 5. 验证与测试计划

### 5.1 模拟端测试
1. **MQTT 代理环境**：本地部署 EMQX 代理，或使用公共测试代理（如 `broker.hivemq.com`）。
2. **设备端模拟器**：编写 Python / Node.js 脚本模拟采样设备，负责定时向 `sewage/device/status` 发送模拟 JSON 报文，以及向 `sewage/device/image` 发送随机 JPG 二进制数据，同时订阅 `sewage/app/settings` 接收并打印参数。

### 5.2 核心功能用例
1. **前后台持久化测试**：App 处于后台，模拟树莓派在通道日志主题（如 `pi_water/log/1`）发布报警消息，检查 Android 状态栏是否能正常弹出 Notification，并在重新打开 App 后能在日志列表中完整看到该条图文信息。
2. **离线重连测试**：在 App 运行中开启手机飞行模式，观察日志控制台是否准确打印重连等待日志。关闭飞行模式后，观察是否在 10 秒内自动恢复连接。
3. **参数同步一致性测试**：在参数设置页修改通道 1 的“水泵动作时间”为 `45` 秒，点击同步，观察树莓派在 `pi_water/config/pump_time/1` 主题接收到的负载值是否为 `45.0`。
